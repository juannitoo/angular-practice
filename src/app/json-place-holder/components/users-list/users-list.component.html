Impossible de contacter le serveur jsonplaceholder. Voilà le message renvoyé si présent :
" {{ error | async }} "

Faire apparaitre ce message d'erreur n'a pas été simple a cause du montage reactif. Le subject n'a pas fonctionné, mais un behaviorSubject oui. Enfin il a fallu retourner le message d'erreur sous forme d'obsevable dans le template car il est issu d'un flux. Vous pouvez vous en apercevoir en rafraichissant la page une fois une erreur créée, par exemple en modifiant l'url vers l'api. En rafraichissant la page, nous ne sommes plus abonnés, et donc le message n'apparait plus.

Je ne sais pas si c'est le meilleur des montages où s'il faudrait passer par un autre lifecyclehook que le ngOnInit, mais en l'état ca fonctionne en cas d'erreur.
