Démo avec Angular

Ce repository vous permet d'évaluer un peu mieux mon niveau avec le framework Angular. Ce front-end est déployé sur l'infrastructure serverless AWS S3 via un pipeleine Devops CICD github Actions pour que vous puissiez l'essayer. Le back-end NodeJs est déployé manuellement sur AWS ECS qui permet le déploiement de conteneur Docker sur une infrastructure serverless, qui n'est pas le synonyme de simplicité lors de la mise en route. Ce n'est pas aussi simple que les fonctions lambda. Enfin, la base de données est un base NOSQL Mongo Atlas également déployé sur AWS mais gérée via Mongo-Atlas et non directement sur AWS. Voilà le lien vers ce repository Angular.

J'ai pour l'instant créé 3 features modules, jsonPlaceHolder, Json-server, et NodeJs qui sont des sources de datas distinctes. Je suis en ce moment sur la partie NodeJs qui n'est pas terminée, et sur laquelle j'ai pour l'instant mis en place une authentification via jsonWebToken.

JSONPlaceholder est une API REST gratuite de démo qui permet de récupérer des datas, et de faire un simili CRUD. J'ai mis en place une architecture modulaire et lazy loadé ce module, mais aussi un réactive state. C'est la partie la plus aboutie.

Json-server est un module node qui permet de réellement mettre à jour ou supprimer les datas que je vais envoyer via mes requetes http, et non plus seulement faire du fake en console. Il faut donc avoir un serveur qui tourne en parallèle. Les datas ont été récupérées sur l'api jsonplaceholder pour initialiser le serveur.

NodeJs me permet de réaliser une petite API Rest avec Mongo et Express pour avoir de la persistance dans les datas, et qui me permet de justifier du côté full-stack de mon profil, en plus de Python Django PostgreSql durant 3 ans ! J'ai commencé à mettre en place ici une authentification JWT.
